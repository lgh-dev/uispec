# UISpec Create - 创建设计规范

根据用户需求描述，生成一套通用的 UI 设计规范。

## 规范维度

生成的规范包含以下 6 个维度：

1. **视觉风格** - 色彩、质感、氛围
2. **布局** - 间距、网格、响应式
3. **交互** - 状态、反馈、操作
4. **动效** - 过渡、时长、缓动
5. **字体** - 家族、大小、层级
6. **图标** - 风格、尺寸、使用

## 规范格式

生成的设计规范保存为 `.uispec/specs/{规范名}.md`，包含：

```markdown
# {规范名称}

## 1. 视觉风格

### 色彩
- 主色: #xxx
- 辅助色: #xxx
- 中性色: #xxx

### 质感
- 背景色: #xxx
- 表面色: #xxx
- 边框/分隔: #xxx

## 2. 布局

### 间距系统
- 基础单位: 4px
- 间距阶梯: xs, sm, md, lg, xl, 2xl

### 响应式断点
- sm: 640px
- md: 768px
- lg: 1024px
- xl: 1280px

## 3. 交互

### 状态
- hover
- active
- focus
- disabled

### 反馈
- 成功、警告、错误、成功

## 4. 动效

### 时长
- fast: 150ms
- normal: 200ms
- slow: 300ms

### 缓动
- ease-out (进入)
- ease-in (离开)
- ease-in-out (变化)

## 5. 字体

### 字体家族
- 衬线/无衬线
- 等宽字体（代码场景）

### 字号层级
- xs, sm, base, lg, xl, 2xl, 3xl...

## 6. 图标

### 风格
- 线框/实心/双色

### 尺寸
- sm: 16px
- md: 20px
- lg: 24px

## 80/20 关键规则

设计规范的关键规则总结（核心要点提炼）

## 执行流程

### 第一步：分析需求
理解用户对设计规范的需求描述，提炼关键要素。

### 第二步：生成规范
根据需求生成通用的 6 维度设计规范：
- 视觉风格 - 色彩、质感
- 布局 - 间距、断点
- 交互 - 状态、反馈
- 动效 - 时长、缓动
- 字体 - 家族、层级
- 图标 - 风格、尺寸

### 第三步：保存规范
将规范保存到 `.uispec/specs/{规范名}.md`

### 第四步：更新配置
更新 `.uispec/current-spec.json` 指向新规范

## 示例

```
/uispec-create 现代简约的移动端设计规范
/uispec-create 深色主题的数据可视化界面
/uispec-create 适合后台管理系统的设计规范
```

## 注意事项

- 只生成 6 个维度的规范内容，不要添加组件规范
- 保持规范通用，避免特定领域术语
- 80% 的效果即可，不追求完美
- 确保规范之间的一致性
- 使用常见的色彩和数值
