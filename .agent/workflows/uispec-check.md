# UISpec Check - 规范审查

审查已开发的功能是否符合当前选定的 UISpec 设计规范。

## 审查项目

我会按照以下项目进行审查：

### 1. 色彩系统
- 是否使用了规范定义的色彩
- 色彩对比度是否满足可访问性要求
- 品牌色使用是否恰当

### 2. 间距系统
- 元素间距是否符合规范
- 内边距和外边距是否一致
- 留白是否充足

### 3. 字体排版
- 字体大小是否符合规范
- 行高和字重是否正确
- 文本层级是否清晰

### 4. 组件样式
- 组件样式是否符合规范
- 交互状态是否完整
- 动画效果是否正确

### 5. 响应式设计
- 移动端适配是否良好
- 断点设置是否合理
- 触摸目标大小是否足够

## 执行流程（必须严格按顺序执行）

### 第一步：读取当前规范配置
从 `.uispec/current-spec.json` 读取当前选定的规范名称。

### 第二步：读取完整规范文件（关键！）
**CRITICAL**: 必须使用 Read 工具读取完整的规范文件，不能使用 limit 参数！

正确示例：
```javascript
Read('.uispec/specs/dark-elegant.md')  // ✅ 读取全文，不使用 limit
```

错误示例：
```javascript
Read('.uispec/specs/dark-elegant.md', limit: 100)  // ❌ 只读部分
```

**必须读取完整规范后才能进行审查！**

### 第三步：审查代码
根据完整规范对提交的代码进行全面审查。

请提供需要审查的代码或页面，我会给出详细的审查报告。
